项目说明文档

小组成员：钟志东、黄一杨、王舒晗

1. 可视化方案与核心问题

本项目旨在通过"交互式数据新闻"的形式，以递进式的视觉叙事回答关于地震的一系列核心问题。从物理机制的深层理解，到生活场景中的实际应对，每一个章节都拓展了对地震灾害的认识维度。

具体而言，本项目通过四个章节回答四类问题：首先揭示地震如何发生与传播——通过板块构造理论和地震波物理，让观众理解这一自然现象的科学根源；其次阐明震级与烈度如何直观影响人类居所——通过建筑模拟展示不同级别地震的破坏程度；再次呈现全球地震的空间分布、时间规律与震级统计——揭示地震的地理集中特征，与第一章的板块理论形成呼应，深化对地震特点的认识；最后提供个体与集体在面对地震时的应对策略——从预警技术到避险方法，阐述如果我们遭遇地震，为什么要抓紧宝贵的时间先躲避后撤离，如何通过科学知识将恐惧转化为行动力。

2. 设计决策与替代方案

本项目在可视化编码与交互技术的选择上，经历了多轮方案比较与论证。

  • 视觉编码：在色彩选择上，我们有意抛弃了传统的"红黄警示色"，转而采用高级的灰蓝+沙金配置（#E8EDF2 / #C2A56D），这种冷调的配色方案赋予了整个项目纪实新闻般的冷峻感，更好地传递了数据的客观性。将地震波统一抽象为"点击生成的同心圆"这一视觉元素，贯穿全文，形成强有力的视觉记忆点。
  • 交互技术：采用了"Scrollytelling"线性叙事结合按需交互（点击/拖拽/滑块）的混合模式。替代方案曾考虑采用"完全由用户自主探索的Dashboard式设计"，但鉴于本项目具有科普与教育的明确意图，线性叙事引导+关键节点交互（如拖拽小人避险、点击产生地震波）的组合能提供更深层次的沉浸感和学习效果。
  • 首尾呼应设计：在第一章和第四章（地震预警部分）统一使用"点击产生地震波、观察波的传播"的交互模式，强化了整个叙事弧线的完整性与统一性。

4. 开发流程与分工评述

本项目采用 Git 版本控制管理，建立了清晰的特性分支结构（feature branches），每个章节与功能模块对应独立分支，确保团队协作的高效性与代码的可追溯性。最终通过代码审查（code review）合并至主分支，保证代码质量。

开发耗时与关键瓶颈：整体团队耗时约 98 工时。其中，D3.js 与物理坐标系的动画绑定、地图数据的平滑重构与多层级重绘，以及避险场景的交互逻辑实现是耗时最多的环节。

分工详情：

  • 钟志东：整体架构设计、第一章物理波动、叙事文案与说明文档整理。
  • 黄一杨：第二章烈度互动实现与第三章全球数据可视化与前端部署。
  • 王舒晗：第四章避险交互与地震波传播速度。

在开发过程中，团队通过定期同步会议、代码审查与问题回溯，有效应对了跨模块集成、性能优化等技术挑战。Scrollytelling 的实现需要精细调控滚动事件与 D3 动画的同步，这要求团队在前期设计阶段充分沟通；地球地图的多层渲染（背景图层、板块边界层、地震点层）也经历了多次迭代，以达到视觉清晰度与交互流畅度的平衡。这些挑战最终转化为项目的核心竞争力——用科学严谨的方式讲述一个既震撼又易懂的故事。
