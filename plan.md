# 全球地震与灾害可视化 - 内部开发指南 (2天冲刺版)

## 1. 项目概况与开发目标

*   **项目形态**：单页滚动交互数据新闻 (Scrollytelling)。
*   **开发周期**：即日起至 5.9上午（如果我们当天就要展示的话，可能性不大）或者5.10，总计约 48 / 72小时。
*   **核心技术**：原生 JS + Vite + D3.js (v7)。不引入 Vue/React，保持轻量。
*   **核心基调**：冷淡、客观、高级的数据新闻风。避免大红大黄，全局严格使用规范设定的 4 种颜色。

## 2. 协作与 Git 规约 (必读)
为避免在2天的极限时间内产生严重的合并冲突，本项目实行**“按文件物理隔离”**的开发模式：
1. **各司其职**：每个人只在自己负责的 `chX.js` 脚本和自己对应区块的 HTML/CSS 中写代码。
2. **禁止跨文件修改**：如果需要调用别人的功能（例如点击震动特效），调用对方暴露出来的函数，**不要直接去改对方的文件**。
3. **每日同步**：每天合适时间进行代码合并（Merge），组长负责解决冲突并推送到主分支。

## 3. 全局规范指引
*   **设计规范**：详见仓库中的 `skill.md`。
    禁止使用高饱和度的纯红纯黄。在 CSS `:root` 中严格定义并使用以下变量：
    - `--bg-color`: `#E8EDF2` (浅灰蓝，作为页面唯一主背景色)
    - `--text-main`: `#000000` (黑色，用于全局正文、普通段落、常规坐标轴文字)
    - `--text-highlight`: `#2C3947` (深板岩色，用于重要文字强调、数据指标高亮、各级核心标题)
    - `--accent-blue`: `#547A95` (钢蓝色，用于次要交互、P波(纵波)可视化、安全状态指示、常态数据散点)
    - `--highlight-gold`: `#C2A56D` (沙金色，用于核心警示、S波(横波)可视化、危险状态指示、核心交互热区)
*   **AI 辅助规范**：让 AI 写代码前，**必须**把 `skill.md` 的内容发给它，作为上下文预设，防止 AI 乱加库或用错颜色。

## 4. 详细分工与模块职责

> 备注：需要什么功能但是组员没有提供合适的接口的时候，就让ai重新写一个在自己的部分用就可以了

### 👨‍💻 成员 A 钟志东：组长 (架构控制、物理属性、构建发版)

**核心职责：搭架子、负责物理属性可视化与基础 CSS 动效。**

*   **工程基建**：使用 Vite 初始化项目，搭建 `index.html` 的四个 section 骨架，定义好全局 CSS 变量。
*   **公共逻辑 (`main.js`)**：编写 `IntersectionObserver` 监听器。当用户滚动到对应章节时，触发对应组员写的初始化/动画函数。
*   **公共基建 (`utils.js`)**：开发 `createSeismicWave(x, y)` 函数。实现点击某处后生成扩散的同心圆 SVG 动效（D3 `transition` 实现），供所有人调用。
*   **章节1 开发 (`ch1-wave.js`)**：
    *   使用 D3 绘制简单的二维坐标系。
    *   使用正弦波/密部疏部动画直观表现横波（S波）和纵波（P波）的区别。
*   **发版部署**：统筹最终测试，配置 `vite.config.js` 相对路径，推送到 GitHub Pages。

### 🧑‍💻 成员 B 建议黄一杨：房屋抖动/坍塌效果、作业2的复用整合

**核心职责：最核心的真实数据视图（作业2复用）。**

*   **章节2 开发 (`ch2-intensity.js`)**：
    *   使用简单的 SVG 绘制 3-4 种不同结构的房屋。
    *   实现一个 HTML `<input type="range">` 震级滑块。
    *   监听滑块数值，动态修改房屋 DOM 的 CSS `animation-duration` (控制抖动频率) 和 `transform` (模拟倾倒或掉落)。
*   **章节3 开发 (`ch3-global-data.js`)**：
    *   将之前作业2的代码平移过来。
    *   移除之前点击切换Tab的逻辑，改为上下响应式排列（地图 -> 散点图 -> 柱状图）。
    *   确保三表联动的逻辑在单页结构下正常运行。

### 👱‍♂️ 成员 C 建议 王舒晗：交互游戏、叙事过渡与文档统筹

**核心职责：负责AI文案生成、页面过渡衔接及拖拽/透视等轻交互。**

*   **章节4 开发 - 避险微游戏 (`ch4-evacuation.js`)**：
    *   使用 `d3.drag()` 实现火柴人的拖拽逻辑。
