# 项目上下文：全球地震与灾害程度交互式可视化文档

## 1. 设计思路与叙事结构
本项目采用“滚动叙事 (Scrollytelling)”模式，旨在通过交互式图表解答地震的成因、破坏力表现、全球分布现状及防震减灾措施。
整体设计要求保持客观冷静的“数据新闻”风格，避免过度使用警示色制造恐慌感。通过一致的“波纹交互”线索贯穿全文。

叙事分为四个递进章节：
- **Chapter 1 (物理学基石)**：解析地震波（P波与S波）的物理传播形式与差异。
- **Chapter 2 (灾害评估)**：建立震级（能量释放）与烈度（结构破坏）的映射关系。
- **Chapter 3 (全球现状)**：基于实时客观数据（USGS），映射全球地震在空间、时间与震级上的分布特征。
- **Chapter 4 (防灾减灾)**：基于物理学视角的个体避险（空间结构分析）与工程防震原理（隔震与抗震体系）。

## 2. 全局色彩规范 (Color Palette)
禁止使用高饱和度的纯红纯黄。在 CSS `:root` 中严格定义并使用以下变量：
- `--bg-color`: #E8EDF2 (浅灰蓝，作为页面唯一主背景色)
- `--text-main`: #000000 (黑色，用于全局正文、普通段落、常规坐标轴文字)
- `--text-highlight`: #2C3947 (深板岩色，用于重要文字强调、数据指标高亮、各级核心标题)
- `--accent-blue`: #547A95 (钢蓝色，用于次要交互、P波(纵波)可视化、安全状态指示、常态数据散点)
- `--highlight-gold`: #C2A56D (沙金色，用于核心警示、S波(横波)可视化、危险状态指示、核心交互热区)

## 3. 技术栈选型
- **构建工具**：Vite
- **核心逻辑**：原生 JavaScript (ES6+ Module)
- **可视化引擎**：D3.js (v7)（负责坐标轴绘制、波形生成、散点与柱状图联动、拖拽交互）
- **地理数据解析**：TopoJSON Client
- **样式方案**：原生 CSS (依赖 CSS Variables 和 Keyframes 实现基础物理动效)
- **滚动监听**：原生 IntersectionObserver API

## 4. 核心线索交互 (Core Motif)
全文需复用统一的“地震波生成函数”。当用户点击特定坐标点时，通过 D3.js 生成同心圆 SVG 元素，利用 `transition` 实现半径放大与透明度衰减，模拟地震波传播。

## 5. 项目结构与文件作用规划
```text
earthquake-viz/
├── index.html               # 页面骨架：按顺序包含 ch1 到 ch4 的 section，以及底部的 doc 模块。
├── style.css                # 全局样式表：定义色彩变量、排版重置、全局动画帧(@keyframes)。
├── main.js                  # 入口文件：注册 IntersectionObserver，初始化各章节模块。
├── data/                    # 静态数据目录
│   └── pb2002_boundaries.json # 板块边界数据
├── scripts/                 # 业务逻辑脚本目录
│   ├── utils.js             # 公共工具：地震波形生成函数(createSeismicWave)、日期格式化等。
│   ├── ch1-wave.js          # 章节1逻辑：D3绘制横波(S波)/纵波(P波)坐标系及动画。
│   ├── ch2-intensity.js     # 章节2逻辑：监听震级滑块，结合CSS控制房屋SVG形变与破坏动效。
│   ├── ch3-global-data.js   # 章节3逻辑：封装世界地图、散点图、柱状图的三表数据获取与联动筛选。
│   ├── ch4-evacuation.js    # 章节4逻辑：d3.drag拖拽小人避险判定，以及建筑抗震机制剖面图的交互逻辑。
└── package.json             # Vite 及相关依赖配置
```
## 6. 开发规范

解耦：每个模块（ch1~ch4）必须对外暴露唯一的初始化函数（如 export function initCh1() {}）。
性能：动画仅在元素进入可视区域（IntersectionObserver 触发）时执行，离开可视区域后需暂停或销毁冗余的 DOM 元素。
